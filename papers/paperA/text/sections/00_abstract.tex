\begin{abstract}
Hardware-oriented neural-graph programs are hard to validate: small numerical or scheduling differences between an interpreter, a C++ reference, and an HLS/Vivado flow can silently change results. NEMA addresses this by defining deterministic tick semantics with an explicit snapshot rule, fixed-point arithmetic (RNE rounding and saturating overflow), and a bit-exactness criterion based on per-tick SHA-256 digests of packed voltages. We implement NEMA as a supporting DSL/IR toolchain and provide scripts that execute the same programs through golden simulation, a C++ reference, Vitis HLS (csim/csynth), and (where applicable) Vivado implementation, emitting machine-readable \texttt{bench\_report.json} outputs. On three core benchmarks (B1, B2, B3) the verification harness reports \texttt{mismatches\_len=0} and \texttt{digestMatchOk=true} over 20 ticks, and an independent checker reproduces the same outcomes. For the same benchmarks, the hardware pipeline completes through Vivado implementation and captures QoR and timing with non-null WNS. The artifact pack freezes all tables, logs, and checksums needed to reproduce these claims.
\end{abstract}
