# NEMA Deploy (Best Effort)

This document covers the minimum path from a generated `.bit` to a board program attempt.

## Preconditions

- `vivado` available in `PATH`.
- Target board connected via JTAG or network hardware server.
- Valid bitstream artifact generated by NEMA (`hardware.vivado.bitstreamPath` in `bench_report.json`).

## 1) Generate a bitstream from B1

```bash
python -m nema vivado bitstream example_b1_small_subgraph.json \
  --outdir build_hw/bit_b1 \
  --ticks 2 \
  --part <your_part>
```

Expected output:

- `build_hw/.../*.bit`
- `build_hw/.../bench_report.json` with:
  - `hardware.vivado.part`
  - `hardware.vivado.bitstreamPath`

## 2) Deploy (best effort)

```bash
bash tools/fpga/deploy_bitstream.sh --bit build_hw/bit_b1/<modelId>/hw_reports/vivado_batch/nema_kernel.bit
```

Optional flags:

- `--hw-server <url>`: pass explicit hw server endpoint
- `--target <filter>`: pass Vivado `open_hw_target -filter` selector

## Notes

- This script does not guarantee success without a connected/compatible board.
- It intentionally does not claim board-level validation beyond the Vivado Hardware Manager programming attempt.
